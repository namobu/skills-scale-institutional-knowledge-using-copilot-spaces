# OctoAcme Project Management Docs

This repository contains OctoAcme's project management process documents. Use this README as your entry point to understand when and how to apply each process, key artifacts, and team rhythms.

## Overview

OctoAcme follows a structured, iterative project management approach centered on five phases: **Initiation** (validating business need and stakeholder alignment), **Planning** (breaking work into shippable increments), **Execution & Tracking** (daily delivery with continuous monitoring), **Release & Deployment** (standardized, risk-managed deployment), and **Retrospective & Continuous Improvement** (capturing learnings). Our approach emphasizes customer-first principles, clear ownership, data-informed decisions, iterative delivery, and psychological safety. Key artifacts include the Project One-pager, backlog with acceptance criteria, risk register, and release notes. Team rhythms include daily standups, weekly syncs, demos, and retrospectives.

## Documents

- **[Project Management Overview](./octoacme-project-management-overview.md)** — Concise introduction to OctoAcme's approach, core roles (PM, Product Manager, Developers, QA), key artifacts, and high-level lifecycle. Start here for a quick orientation.

- **[Project Initiation Guide](./octoacme-project-initiation.md)** — Steps to validate and authorize new work. Includes the Project One-pager template (problem, goal, success metrics, stakeholders, timeline, risks) and a go/no-go decision gate before moving to planning.

- **[Project Planning](./octoacme-project-planning.md)** — Turn an approved initiative into an actionable plan. Covers backlog creation, estimation, definition of done, dependency mapping, and release planning.

- **[Execution & Tracking](./octoacme-execution-and-tracking.md)** — Manage day-to-day delivery. Includes team rhythm (standups, weekly syncs, demos), GitHub Projects workflow, PR standards, quality & testing practices, and blocker escalation paths.

- **[Risk Management & Communication](./octoacme-risks-and-communication.md)** — Identify, assess, and mitigate risks using a Risk Register. Covers stakeholder communication templates, incident response, and escalation paths (team → PM → Product Lead → Sponsor).

- **[Release & Deployment](./octoacme-release-and-deployment.md)** — Standardize how OctoAcme releases features to production. Includes release types (patch, minor, major), pre-release requirements, deployment checklist, and rollback playbook.

- **[Retrospective & Continuous Improvement](./octoacme-retrospective-and-continuous-improvement.md)** — Capture learnings after each sprint, release, or milestone. Covers retrospective structure, action item tracking, and how to measure and celebrate improvements.

- **[Roles & Personas](./octoacme-roles-and-personas.md)** — Role summaries and responsibilities for Developers, Product Managers, and Project Managers, including their goals and typical communication patterns.

## How to Use This Documentation

- **Getting Started**: Read [Project Management Overview](./octoacme-project-management-overview.md) first, then choose the process doc that matches your current phase (Initiation, Planning, Execution, Release, or Retrospective).

- **Copilot Spaces**: Reference these docs in Copilot Spaces to ground your knowledge in OctoAcme's processes. Add this folder to a Space's context for role-specific guidance.

- **Keeping Docs Current**: When adding or updating process documentation, update this README to keep summaries and links current. Use the [Add Content to Project Management Process Docs](../.github/ISSUE_TEMPLATE/add-update-content-to-process-docs.yml) issue template to propose changes.

- **Onboarding**: Link this README from the project root and reference it in team onboarding materials so new members understand OctoAcme's project management practices.
