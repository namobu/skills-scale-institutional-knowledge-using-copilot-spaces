# Project Process Checklist (template)

Use this checklist during Initiation and maintain it through Execution.

Project metadata
- Project name:
- Owner / Sponsor:
- Named role-holders:
  - Product Manager:
  - Project Manager:
  - Technical Lead:
  - Engineering Manager:
  - Release Manager:
  - QA Lead:
  - SRE / On-call:
  - Design Lead:
  - Data Analyst:

Initiation (complete before planning)
- [ ] Project one-pager created (use docs/project-onepager-template.md)
- [ ] Stakeholders identified and communication plan set
- [ ] Success metrics defined and instrumentation owner assigned
- [ ] High-level timeline and milestones documented
- [ ] Initial risk register entry created

Planning
- [ ] Backlog prioritized and acceptance criteria written
- [ ] Definition of Done documented
- [ ] CI and test strategy defined (unit, integration, smoke)
- [ ] Release plan drafted with rollback strategy and owner (Release Manager assigned)

Execution
- [ ] PRs follow PR & CI guidance (docs/pr-and-ci-guidance.md)
- [ ] Automated tests pass in CI before review requested
- [ ] Weekly risk review and update to risk register
- [ ] Demos scheduled for milestone deliveries

Release readiness
- [ ] All acceptance criteria met and QA sign-off
- [ ] Smoke tests prepared and owner assigned
- [ ] Rollback plan and verification steps documented
- [ ] Stakeholders notified per communication plan
